1 搭建环境

2 开始工程开发

2.1 下载工程

 
$ cd /d
$ mkdir worksapce
$ cd worksapce
$ git clone git@github.com:derekkuo/TechnicalLibrary.git

2.2 创建自己的开发分支，进行开发

 
$ cd TechnicalLibrary/
$ git branch dev-someone
$ git checkout dev-someone
$ git branch -v
* dev-someone de59645 Merge branch 'dev-derekkuo'
  master      de59645 Merge branch 'dev-derekkuo'

根据项目任务要求，开发自己的模块。

3 更新项目

3.1 更新下载master主分支

更新下载项目的主分支，有fetch更新和pull更新两种方式，以下详细介绍。

方式一：fetch抓取更新，再手动合并

 
$ git checkout master
$ git fetch origin master
$ git diff FETCH_HEAD
$ git merge FETCH_HEAD

建议分支开发人员使用fetch更新方式，这样在每次更新时，可详细了解每次更新的内容。

方式二：pull直接下载自动合并更新

 
$ git checkout master
$ git pull origin master
注意

应该经常或定期的从服务器中更新项目，下载到最新的代码，可以了解到整个项目的开发进度。

3.2 合并master到自己的开发分支，继续自己分支开发

 
$ git checkout dev-someone
$ git merge master

3.3 合并时，冲突的处理

在合并分支时，如出现对同一个文件同一部分进行修改，会产生合并冲突，需要手工进行修改后，再提交文件。

$ git merge master
Auto-merging README
CONFLICT (content): Merge conflict in README
Automatic merge failed; fix conflicts and then commit the result.

此时打开冲突的README文件，会在相应冲突的部分出现如下内容。

<<<<<<< HEAD
项目说明
=======
这个项目
>>>>>>> refs/heads/master

此时需要手工的进行合并，将文件中冲突标记删除，修改内容至正确后，手工add添加修改后的文件，再commit提交文件。

$ git add README
$ git commit README

4 推送自己的开发分支到服务器

完成阶段性工作并且测试无误后，推送自己的开发分支到服务器。

4.1 从服务器更新合并，以避免管理员在合并时出现冲突

 
	$ git checkout master
	$ git pull origin master
	$ git checkout dev-someone
	$ git merge master

4.2 推送自己的开发分支

 
	$ git push origin dev-someone
注意

开发人员在推送时，不可推送本地的master主分支到远程服务器。

	X $ git push origin master //分支开发人员，不可直接上传合并master主分支
	X $ git push 	//不可将本地所有分支推送到远程服务器

将自己的开发分支推送到服务器后，管理员会合并你的开发分支，同时你也还是可以继续向自己的开发分支中添加更新。 分支合并后，管理员会将你的开发分支删除，如想自己删除分支，可使用如下命令。

	$ git push origin :dev-someone
