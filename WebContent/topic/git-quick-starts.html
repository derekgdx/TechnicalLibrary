2 开始工程开发

2.1 下载工程

 
$ cd /d
$ mkdir worksapce
$ cd worksapce
$ git clone git@github.com:derekkuo/TechnicalLibrary.git

2.2 创建自己的开发分支，进行开发

 
$ cd TechnicalLibrary/
$ git branch dev-someone
$ git checkout dev-someone
$ git branch -v
* dev-someone de59645 Merge branch 'dev-derekkuo'
  master      de59645 Merge branch 'dev-derekkuo'

根据项目任务要求，开发自己的模块。

2.3 测试无误后，推送自己的开发分支到服务器

 
$ git push origin dev-someone

要注意的是，在推送时，不可推送本地的master主分支到远程服务器。

X $ git push origin master //不可这样写命令，分支开发人员，不可直接上传合并master主分支
X $ git push 	//不可这样写命令，不可将本地所有分支推送到远程服务器

3 更新项目

3.1 更新下载master主分支

更新下载项目的主分支，有fetch更新和pull更新两种方式，以下详细介绍。

方式一：fetch抓取更新，再手动合并

 
$ git checkout master
$ git fetch origin master
$ git diff FETCH_HEAD
$ git merge FETCH_HEAD

建议分支开发人员使用fetch更新方式，这样在每次更新时，可详细了解每次更新的内容。

方式二：pull直接下载自动合并更新

 
$ git checkout master
$ git pull origin master
注意

应该经常或定期的从服务器中更新项目，下载到最新的代码，可以了解到整个项目的开发进度。

"经常push和fetch"。

3.2 合并master到自己的开发分支，继续自己分支开发
